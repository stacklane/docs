Manifest

The manifest configuration file defines basic properties about a site.

The manifest is the only required configuration file in a site's source. It should be in the root directory and named /ðŸŽ›.yaml. Only a single field is required -- all other fields are optional: 


name: My Site or Web App Name

name

The name of the site / application as it is usually displayed to the user (e.g., amongst a list of other applications, or as a label for an icon). Required.

domain

true/false value that determines whether a site may exist at a standalone domain name. Default: true

path

true/false value that determines whether a site may be nested at sub-path of another site. Default: true

frameable

true/false value that determines whether a site may be embedded in a frame of another site on the same domain. Default: false
