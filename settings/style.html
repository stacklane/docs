Styles

Styles are a purpose specific SCSS configuration file.

The style configuration file is a plain SCSS file that accepts values/variables only. Unlike regular SCSS variables, these values may be overridden when nesting the site, and also accessed from additional locations (server-side JavaScript, Mustache templates, and of course imported into other SCSS files).

File Format

Create a file at the root /ðŸŽ¨.scss to define your properties. This is a limited SCSS file for variables only. The following SCSS data types are supported: string, boolean, color (hex/rgb/preset), url. URLs must be absolute and point to static images contained in the source.

Variable Naming

As described below, the variables may be used in several different environments/languages. Therefore it's important to understand how naming your variables gets translated. The simplest thing to do is to use "camelCase" which ensures that your variables are identical across all languages.

However a more SCSS/CSS friendly convention is to use dashes for variables names such as $color-primary. If you choose this latter convention, simply be aware that from JS and Mustache that same variable is accessed with an underscore: color_primary

SCSS Access

From another SCSS file, access variables defined in the style file just as you would any other SCSS import: 


@import '/ðŸŽ¨';

body{
  background-image: $image-background
}

Mustache Access

From a mustache file import the style properties. Note that url(...) values are unwrapped to their paths only. 


<!--TEMPLATE mustache-->
{{% import * as Style from 'ðŸŽ¨'}}

<img src="{{Style.image_background}}">

JavaScript Access

From a server side JS file import the style properties: 


import * as Style from 'ðŸŽ¨';

if (Style.some_boolean_option){
   ...
}

Advanced Configuration

Consider using boolean and string values to provide advanced configuration or "modes" to a site. Because the values are accessible from all locations, you can go deeper than CSS alone. For example in /ðŸŽ¨.scss: 


$myMode: true;

Then in a Mustache endpoint: 


<!--TEMPLATE mustache-->
{{% import * as Style from 'ðŸŽ¨'}}

{{#Style.myMode}}
In special mode.
{{/Style.myMode}}
